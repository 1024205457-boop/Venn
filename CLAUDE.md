# Venn AI - 开发规范

## 技术栈
Next.js 16 + React 19 + D3.js 7 + TailwindCSS 4 + better-sqlite3 + 智谱 GLM API

## 维恩图交互规范

### 新增节点（虚线圆 +）
- 每层子节点列表末尾都有一个虚线圆 `+` 按钮，用于新增同级节点
- 没有 children 的节点（包括一级叶子节点、二级叶子节点）内部也要有虚线圆，允许用户添加第一个子节点
- 交叉区域（lens）内也有虚线圆，点击后向 relation.sharedConcepts 追加新概念
- 虚线圆大小必须和同级圆一致，不能缩小（方便点击）
- 虚线圆样式：半透明填充（fill-opacity 0.2-0.3）+ 虚线描边 + "+" 文字，hover 时加深
- 虚线圆要有透明的更大点击区域（1.3 倍半径），确保容易选中
- 计算 zoom 视口范围（childCenter.spread）时必须把虚线圆位置算进去，确保放大进入后虚线圆直接可见，不需要额外拖拽

### 新增节点后的视图保持
- 点击 `+` 新增节点后，必须保持当前的缩放位置和层级，不能闪退回上一层
- 通过 savedTransformRef 保存当前 zoom transform，重绘后恢复
- 新节点应在当前视图中就地出现，布局平滑重排

### 深入分析（嵌套生成）
- 选中任意节点后，selection panel 中有"深入分析"入口
- 可粘贴文本或上传文件（.txt/.md/.docx），调用 AI 分析后将结果作为该节点的 children 追加
- 分析过程中显示 loading 状态，完成后保持当前视图位置

### 两种分析模式
- **收集模式**：用户输入关键词/一级标题，AI 运用知识展开生成子概念和交叉关系。无上传入口
- **整理模式**：用户粘贴已有素材或上传文件，AI 严格忠于原文提取结构，不自行扩展。支持 .txt/.md/.docx，最大 5 万字，上传后转为纯文本
- 用户可选择生成层级（2层或3层），仅控制 AI 生成深度
- 所有层级的虚线圆（+按钮）都始终存在，用户可手动新增任意层级的子节点

### 缩放层级
- 简洁模式：渐进式 zoom 显露（CHILD_SHOW_SCALE=2.5, GRANDCHILD_SHOW_SCALE=6）
- 透视模式（原边框模式）：所有层级始终可见
- 双击放大进入子级，双击空白区域退回上一级

### 卡片面板布局（selection panel）
- 交叉区域和普通节点使用同一个 selection panel，不用浮动 tooltip
- 卡片顺序：标题 → 子概念列表 → 笔记（description 作为默认内容）→ 深入分析（最底部）
- 一级卡片不标注交叉领域（交叉区域的 lens 里已有子标题）

### 交叉区域渲染
- 两两交叉：混合两个父圆的 HSL 颜色，概念圆沿 lens 长轴排列
- 三交叉：使用独立的暖棕色（hsl(40, 25%, 32%)），与两两交叉区分
- 渲染层级（从底到顶）：两两交叉 lens fill → 三交叉 fill → 概念圆（lensConceptOverlay）
- 概念圆 opacity 0.9，font-weight 700，确保清晰可见
- 交叉标题 font-size 12px，font-weight 700，stroke 描边增强可读性
- 存在三交叉时，两两交叉的概念圆沿长轴向外偏移，避开三交叉中心区域

### 布局规则
- 4 个一级圆使用 2x2 网格布局（非环形）
- borders mode 下，有 grandchildren 的子圆居中排列，叶子子圆更小并推向边缘
- AI prompt 要求每个顶层 node 必须有 children（不能为空数组）
